# Contributing to inclusion.md

Thanks for being here. This project exists because inclusion is a moving
target and a collective effort. Contributions are welcome, and contributions
from disabled practitioners and from communities underrepresented in software
tooling are especially valued.

---

## Ways to contribute

- **Language fixes.** Spot ableist, exclusionary, or imprecise language in
  this repo? Open a PR or issue.
- **Domain extensions.** Have a sharp `INCLUSION.md` for healthcare, civic
  tech, fintech, education, hiring, generative tools, gaming, IoT, or
  something else? Add it under `examples/`.
- **Translations.** This file should not be English-only. Translations are
  welcome under `translations/<locale>/INCLUSION.md`.
- **Research updates.** New empirical work on accessibility, representational
  harm, or bias in AI-assisted development? Suggest updates to references.
- **Critique.** This is a draft of an evolving idea. Disagreement with
  framing, structure, or omissions is a contribution. Open an issue.

---

## Ground rules

1. **Center disabled people.** When in doubt, defer to disabled
   practitioners, disability-led organizations, and primary sources from
   disability communities. Cite them.
2. **No inspiration framing.** Disabled users are not motivational fodder.
3. **Identity-first language by default.** "Disabled person," "Autistic
   person," "Deaf person." Defer to specific community or individual
   preference when known.
4. **No "diverse" as a euphemism.** A single person is not "diverse."
   Communities are not "diverse" as a synonym for "non-white" or "non-male."
5. **Don't generate inclusion guidance with an LLM and ship it unedited.**
   That is exactly the failure mode this project is trying to address.
6. **Be specific.** "Make it inclusive" is not actionable. "Default to plain
   language at US grade 6 reading level for consent flows" is.

---

## PR checklist

Before opening a PR, please confirm:

- [ ] The change is consistent with the inclusive language guidance in
      `INCLUSION.md` Section 7.
- [ ] New references are from credible primary sources (research, community
      orgs, standards bodies, disabled practitioners), not just LLM-summarized
      hearsay.
- [ ] New examples in `examples/` are based on real domain knowledge or are
      clearly marked as illustrative starting points.
- [ ] Translations preserve meaning, not just word-for-word substitution.
      Coordinate with someone who has cultural and linguistic context for the
      target locale.
- [ ] You have not added tracking, surveillance, or telemetry to anything.
